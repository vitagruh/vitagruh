# Руководство по системе логирования

## 📋 Обзор

В проекте реализована профессиональная система логирования, соответствующая best practices:

### Ключевые особенности:

1. **RotatingFileHandler** - автоматическая ротация логов при достижении размера
2. **Разделение логов** - ошибки выносятся в отдельный файл
3. **Настраиваемый уровень логирования** - через переменную окружения `LOG_LEVEL`
4. **Детальный формат** - включает время, уровень, модуль, функцию и номер строки
5. **Консольный вывод** - упрощенный формат для быстрой отладки

---

## 🔧 Конфигурация

### Переменные окружения (.env)

```bash
# Уровень логирования: DEBUG, INFO, WARNING, ERROR, CRITICAL
LOG_LEVEL=INFO

# Путь к основному лог-файлу (все уровни от DEBUG)
LOG_FILE=logs/bot.log

# Путь к файлу ошибок (только ERROR и CRITICAL)
ERROR_LOG_FILE=logs/error.log
```

### Уровни логирования

| Уровень | Когда использовать | Пример |
|---------|-------------------|--------|
| `DEBUG` | Отладочная информация | "Получен ответ от API: {...}" |
| `INFO` | Важные события | "Пользователь начал трек билетов" |
| `WARNING` | Предупреждения | "Поезд не найден, повторная попытка" |
| `ERROR` | Ошибки выполнения | "Таймаут запроса к сайту" |
| `CRITICAL` | Критические ошибки | "TOKEN не найден, выход из программы" |

## 📝 Формат записей

### В консоли (упрощенный):
```
2026-04-06 21:16:51 | INFO     | ℹ️ Информационное сообщение
```

### В файле (детальный):
```
2026-04-06 21:16:51 | INFO     | TicketBot | send_welcome:280 | Начат трек для chat_id 12345
```

**Поля формата:**
- `asctime` - дата и время
- `levelname` - уровень логирования
- `name` - имя логгера
- `funcName` - функция, вызвавшая логирование
- `lineno` - номер строки кода
- `message` - само сообщение

## 💡 Примеры использования

### Базовое логирование

```python
from ticket_bot import logger

logger.debug("Отладочная информация")
logger.info("Пользователь подключился")
logger.warning("Предупреждение о проблеме")
logger.error("Произошла ошибка")
logger.critical("Критическая ситуация!")
```

### Логирование исключений

```python
try:
    result = risky_operation()
except Exception as e:
    logger.exception("Ошибка при выполнении операции")
    # Автоматически добавит traceback
```

### Логирование с контекстом

```python
chat_id = message.chat.id
logger.info(f"Начат поиск билетов: from={from_station}, to={to_station}, chat_id={chat_id}")
```

## 🔄 Ротация логов

### Основной файл (bot.log):
- Максимальный размер: **10 MB**
- Количество резервных копий: **5 файлов**
- Общий максимальный размер: **~50 MB**

### Файл ошибок (error.log):
- Максимальный размер: **5 MB**
- Количество резервных копий: **3 файла**
- Общий максимальный размер: **~15 MB**

При достижении лимита:
1. Текущий файл переименовывается (добавляется `.1`, `.2` и т.д.)
2. Создается новый пустой файл
3. Старые файлы удаляются при превышении backupCount

## 🚀 Быстрый старт

1. **Скопируйте шаблон настроек:**
   ```bash
   cp .env.example .env
   ```

2. **Настройте уровень логирования:**
   ```bash
   # Для разработки (подробные логи)
   LOG_LEVEL=DEBUG
   
   # Для продакшена (только важное)
   LOG_LEVEL=INFO
   ```

3. **Запустите тест:**
   ```bash
   python test_logging.py
   ```

4. **Проверьте логи:**
   ```bash
   cat logs/bot.log
   cat logs/error.log
   ```

## ⚠️ Рекомендации

### ✅ Делайте:
- Используйте `logger.exception()` вместо `logger.error()` при обработке исключений
- Логируйте контекст (ID пользователя, параметры запроса)
- Настраивайте `LOG_LEVEL=DEBUG` только при отладке
- Регулярно проверяйте `error.log` в продакшене

### ❌ Не делайте:
- Не логируйте чувствительные данные (токены, пароли)
- Не используйте `print()` для отладки в продакшене
- Не устанавливайте `LOG_LEVEL=DEBUG` в продакшене (будет много шума)

## 🛠 Расширенные возможности

### Добавление своего логгера

```python
import logging

module_logger = logging.getLogger('TicketBot.my_module')
module_logger.info("Сообщение из моего модуля")
```

### Изменение уровня во время выполнения

```python
import logging

logging.getLogger('TicketBot').setLevel(logging.DEBUG)
```

### Фильтрация по модулю

```bash
# Только логи определенного модуля
LOG_LEVEL=WARNING  # скроет DEBUG/INFO/WARNING от всех модулей
```
